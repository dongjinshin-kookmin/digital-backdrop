# 디지털 백드롭 어플리케이션

행사·세미나·협정식 등에서 대형 화면(프로젝터·LED)에 띄우는 **디지털 백드롭**을 누구나 자유롭게 구성할 수 있는 단일 파일 웹 앱입니다. 국민대학교–NKUST MOU 체결식용으로 만든 일회성 백드롭을 일반화하여, 관리자 모드에서 배경·텍스트·로고를 편집하고 저장/공유할 수 있게 했습니다.

외부 CDN·라이브러리에 의존하지 않는 **순수 HTML/CSS/JS 단일 파일**이므로, 로컬에서 더블클릭으로 바로 열거나 GitHub Pages 등 정적 호스팅에 그대로 올릴 수 있습니다.

---

## 빠른 시작

1. `index.html`을 더블클릭하여 브라우저로 엽니다. (또는 웹 서버에 업로드)
2. 처음 열면 **기본 템플릿**(원본 MOU 백드롭을 일반화한 샘플)이 나타납니다.
3. 화면 **우측 상단 모서리에 마우스를 올리면 톱니바퀴 버튼**이 나타납니다. 클릭하면 관리자 편집 패널이 슬라이드 인 됩니다.
   - 키보드 `E` 로도 관리자 모드를 켜고 끌 수 있습니다.
   - `Esc` 로 패널을 닫습니다.
4. 원하는 대로 편집하면 **자동으로 이 브라우저에 저장**됩니다. 다시 열어도 유지됩니다.

---

## 두 가지 모드

| 모드 | 설명 |
|------|------|
| **표시 모드 (기본)** | 백드롭만 전체 화면으로 표시. UI 요소 없음. 프로젝터·대형 화면 출력용. |
| **관리자 모드** | 우측에서 편집 패널이 슬라이드 인. 배경·효과·요소·저장 관리. |

전환: 우측 상단 톱니바퀴 버튼 · `E` 토글 · `Esc` 닫기.

## 기능

### 배경 그라데이션
- **타입**: 방사형(radial) / 선형(linear)
- **각도**(선형) 또는 **중심 위치**(방사형) 조절
- **색상 스톱 2~5개** — 컬러 피커 + 위치(%) 조절, 추가/삭제
- **프리셋 6종**: 국민 로열블루(원본), 미드나잇 잉크, 골든 선셋, 에메랄드, 로열 퍼플, 차콜 그레이

### 장식 효과 (켜기/끄기)
원본 백드롭의 은은한 분위기 요소를 개별 토글로 일반화했습니다.
- 은은한 빛 번짐(sheen) · 도트 그리드 · 대각선 광선(beams) · 떠다니는 입자 · 테두리 프레임

### 텍스트 요소
- 자유롭게 추가/삭제 (행사명, 부제, 일시·장소, 주최 등)
- 편집 항목: 내용(줄바꿈 가능), 글꼴 크기, 굵기(300~900), 색상, **자간(슬라이더, −0.05em ~ 0.5em, 현재값 표시)**, 정렬
- **캔버스에서 드래그로 위치 이동** (관리자 모드에서만)

### 이미지 / SVG
- 파일 업로드(PNG/JPG/SVG) → data URI로 변환하여 요소로 추가
- SVG는 로고·엠블럼 용도로 적합
- 드래그 이동, **핸들 드래그 또는 슬라이더로 크기 조절**, 레이어 순서(맨 앞/맨 뒤), 삭제

### 기관 배치 (로고 가로 배열)
협정식처럼 여러 기관의 로고·기관명을 가로로 나란히 배열하는 전용 기능입니다.
- **기관 수 1~4개** 선택 → 자동으로 가로 중앙 정렬 배치 (1단=중앙, 2단=좌우, 3·4단=균등 분할)
- **구분 기호**: `×` / `·` / `—` / 없음 중 선택 (기관 사이에 삽입)
- **수직 위치**와 **크기 배율** 슬라이더로 전체 블록 조절 (가로 중앙 정렬은 자동)
- **기관명 자간** 슬라이더로 국문·영문 기관명의 letter-spacing 조절
- 기관마다 **로고**(내장 자산 선택 또는 이미지 업로드)와 **기관명 국문/영문**(영문 칸은 `\n`으로 줄바꿈) 편집
- **내장 로고 자산**: 국민대 엠블럼(교표), 국민대 워드마크, NKUST 교표 — 원본 백드롭에서 추출해 앱에 기본 내장. 별도 업로드 없이 바로 선택 가능
- 자유 배치 텍스트/이미지 요소와 **공존**하며, JSON·localStorage에 함께 저장됩니다

### 포토 페이지 (사진 촬영용 두번째 페이지)
행사 기념 촬영용 두번째 백드롭입니다. 사람들이 화면 앞에 서서 사진을 찍으므로, **콘텐츠(타이틀·기관 로고·기관명)를 화면 상단에 모아** 서 있는 사람에게 가려지지 않게 재배치합니다.
- **설정 한 번 → 두 페이지 모두 반영**: 메인 페이지의 배경·장식 효과·기관 배치·타이틀·자유 요소를 **그대로 자동 반영**하되, 상단 집중 레이아웃으로 다시 흐릅니다. 포토 페이지만 따로 편집할 필요가 없습니다.
- **전환 방법**: 키보드 `P`(메인 ↔ 포토 토글) · `1`(메인) / `2`(포토) · 주소 뒤 `#photo` 딥링크 · 관리자 패널 **"포토 페이지"** 섹션의 토글 스위치. 표시 화면에는 전환 버튼이 나타나지 않아 촬영·송출 시 화면이 깨끗하게 유지됩니다.
- **포토 전용 미세 조정**: 콘텐츠 **세로 위치**·**크기 배율** 슬라이더로 상단 집중 정도를 조절

### 저장 / 불러오기
- 모든 상태를 **localStorage에 자동 저장** (새로고침·재실행해도 유지)
- **JSON 내보내기 / 가져오기** — 이미지 data URI까지 포함되므로 다른 PC에서도 동일 화면 재현
- **초기화** — 확인 후 기본 템플릿으로 되돌림

## 대형 화면 대응

- 모든 요소의 위치는 **화면 비율(%) 기반**으로 저장되어, 해상도가 달라도(FHD·4K·LED 월 등) 배치 비율이 유지됩니다.
- 글꼴 크기·이미지 크기는 뷰포트 폭 기준(`vw`) 이므로 화면이 커질수록 함께 커집니다.
- 편집은 일반 모니터에서 하고, 출력만 대형 화면으로 하면 동일한 구도가 확대되어 표시됩니다.

### 편집 → 출력 워크플로 (권장)
1. 사무실 PC에서 편집 후 **JSON 내보내기**.
2. 행사장 출력 PC의 브라우저에서 `index.html`을 열고 **JSON 가져오기**.
3. `E` 로 관리자 패널을 닫고 표시 모드로 전환 → 전체 화면(F11) 출력.

## 키보드 단축키

| 키 | 동작 |
|----|------|
| `E` | 관리자 모드 토글 |
| `P` | 메인 ↔ 포토 페이지 토글 |
| `1` / `2` | 메인 페이지 / 포토 페이지 바로 이동 |
| `Esc` | 관리자 패널 닫기 |
| `Delete` / `Backspace` | 선택한 요소 삭제 (관리자 모드, 입력창 밖) |

## 배포 (GitHub Pages)

**배포 주소**: https://dongjinshin-kookmin.github.io/digital-backdrop/

1. 저장소에 `index.html`을 커밋/푸시합니다.
2. 저장소 **Settings → Pages → Branch** 를 `main`(루트)으로 지정합니다.
3. 발급된 URL로 접속하면 그대로 동작합니다. (별도 빌드 과정 없음)
   - 포토 페이지는 배포 주소 뒤에 `#photo` 를 붙여 바로 열 수 있습니다.

> localStorage와 가져온 JSON은 **접속한 브라우저에 종속**됩니다. 다른 기기·브라우저에서는 JSON을 다시 가져오세요.

## 원본에서 추출한 디자인 요소

기본 템플릿과 "국민 로열블루" 프리셋은 원본 MOU 백드롭에서 다음을 계승했습니다.

- **배경**: 방사형 그라데이션 `radial-gradient(125% 100% at 50% 40%, …)` — 국민대 공식 KMU Blue(`#004f9f`, PANTONE 2728C)를 중심으로 가장자리로 갈수록 짙어짐
  - 스톱: `#0a55a1` → `#004f9f`(44%) → `#042f60`(78%) → `#02172e`(100%)
- **골드 액센트**: `#e6c67d` (kicker·구분선·프레임에 절제되게 사용)
- **장식 효과**: 느리게 흐르는 빛 번짐, 미세한 도트 그리드, 대각선 광선, 떠오르는 입자, 골드 하이라인 프레임
- **타이포그래피**: 넓은 자간의 골드 kicker + 대형 대문자 800 굵기 타이틀
- **로고**: 원본에 내장돼 있던 국민대 엠블럼(교표)·워드마크, NKUST 교표를 그대로 추출해 기본 템플릿의 2단(KMU × NKUST) 기관 배치로 재현

## 알려진 제약 / 참고사항

- **국민대·NKUST 로고는 원본 백드롭에서 추출해 앱에 기본 내장**되어 있으며, "기관 배치" 섹션에서 바로 선택할 수 있습니다. 다른 협력기관 로고는 "이미지 업로드"로 교체하세요.
- **내장 로고는 코드 상수(`BUILTIN_ASSETS`)로만 존재**하고, 저장 상태(localStorage·JSON)에는 참조 키(`logoKey`)만 기록됩니다. 따라서 기본 로고가 저장 데이터에 중복 포함되지 않아 저장 용량이 작습니다(기본 구성 약 3KB). 사용자가 새로 업로드한 로고·이미지만 data URI로 저장됩니다.
- 상태는 브라우저 localStorage에 저장됩니다. **직접 업로드한** 대용량 이미지를 많이 넣으면 브라우저 저장 한도(대략 5MB)를 넘을 수 있으며, 이 경우 저장 실패 안내가 표시됩니다. 무거운 구성은 JSON 내보내기로 백업하세요.
- 이전 버전에서 저장한 데이터는 자동으로 최신 스키마(v3)로 이전(migration)됩니다. 기관 배치가 없던 데이터에는 기본 2단(KMU × NKUST) 구성이 안전하게 채워지고, 구버전에서 기관 로고·기관명이 **자유 배치 요소로 중복** 저장돼 새 기관 배치 레이어와 겹치던 데이터는 자동으로 감지해 정리(중복 요소를 깨끗한 기본 요소로 교체)합니다. 또한 포토 페이지 설정이 없던 데이터에는 기본값이 채워집니다.
- 텍스트 색상은 컬러 피커(HEX) 또는 텍스트 입력(`rgba(...)` 등)으로 지정할 수 있습니다. 컬러 피커는 HEX만 표현하므로 반투명 색은 텍스트 입력을 사용하세요.
- `prefers-reduced-motion` 설정이 켜진 환경에서는 애니메이션(입자 등)이 자동으로 비활성화됩니다.
- 회전·그림자 등 개별 요소의 고급 스타일링은 범위에 포함하지 않았습니다.
